Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

**Answer:

  • The grid search technique is a process of tuning hyperparameters by executing an exhaustive search on a manually specified subset of the hyperparameter space of a learning algorithm. It is carried out to determine the best combination of hyperparamters. A hyperparameter is a parameter that controls the learning process. A grid is created from manual values that can be adopted by hyperparameters e.g. kernel vs C paramter in SVM hyperparamter tuning. The grid search technique is guided by a performance metric, typically measured by cross-validation on the training set. The scenario that exhibits the best performance metric is chosen. This model is then tested with the testing set to ensure the model is adequate and fit-for-purpose.
  • Sets of reasonable discrete values need to be specified for continuous hyperparameters and discrete options (like kernels) are considered too. Grid search then trains the model with each hyperparameter combination and evaluates their performance on a held-out validation set (or by internal cross-validation on the training set). The grid search algorithm outputs the settings that achieved the highest score in the validation procedure. The scenario that exhibits the best performance metric is chosen. This model is then tested with the testing set to ensure the model is adequate and fit-for-purpose.**

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

**Answer: - k-fold cross-validation helps in preventing overfitting. k-fold cross-validation is a resampling procedure used to evaluate machine learning models on a limited data sample. The procedure has a single parameter called k that refers to the number of groups that a given data sample is to be split into. Cross-validation is primarily used to estimate the general model performance when used to make predictions on data not used during the training of the model. 'k' is a parameter that refers to the number of groups a given data sample is to be split into. The general procedure is as follows: 1. Randomise data set 2. Split data set into k groups/folds 3. One fold is dedicated to being the test set and the remainder of the folds make up the training set 4. Train model k times - for each model, use the same data in the same folds, but use a different fold as the test set and the remaining points as the training set 5. The performance metrics reported are then averaged to obtain a final model - To avoid overfitting, hold out part of data as a test set. When evaluating different settings (e.g. hyperparameters) for estimators, there is still a risk of overfitting on the test set because parameters can be ajusted until the estimator performs optimally. To solve this problem, a further part of the training data set can be held out, called the validation set. Training then proceeds on the training set, after which evaluation is done on the validation set. When the experiment seems successful, final evaluation is done on the test set. However, by creating a validation set, the number of samples available for the model to learn from are signifcantly reduced and the results can depend on a particular random choice for the pair of training and validation sets. This problem can be solved by using cross-validation. The validation set is no longer required when doing cross-validation. When using k-fold cross-validation, the training set is split into k smaller sets, where a model is trained using k-1 folds as training data and the resulting model is validated on the remaining fold. The remaining fold is therefore used as a test set to compute a performance measure e.g. accuracy or F1 Score. The model with the best performance measure is chosen. Finally, use the test set to ensure that the chosen model is appropriate.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)
    
    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':list(range(1,11))}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)
    
    # Return the optimal model after fitting the data
    return grid.best_estimator_
